Moon · Level 10 – Databases & SQL for Business

Why databases matter, SQL basics, primary keys and dashboards.

database SQL primary key dashboards
Back to Galaxy

1. Why use a database in business?

A database stores structured data (customers, orders, products) in an organized way.

  • Fast search and filtering.
  • Many users can work at the same time.
  • Data integrity and consistency.

2. Relational database example

MySQL is an example of a relational database. Others: PostgreSQL, SQL Server, Oracle.

3. SQL language

SQL = Structured Query Language – used to communicate with relational databases.

-- get all rows SELECT * FROM customers; -- get only names and cities SELECT name, city FROM customers;

4. Getting data from customers table

A typical query is:

SELECT * FROM customers;

Not TAKE or GET – SELECT is the correct SQL keyword.

5. Primary key choice

A primary key uniquely identifies each row in a table.

  • customer_id is a good primary key.
  • First name, email or city may not be unique.

6. Excel vs database

One big Excel file is not always better:

  • Hard to control changes.
  • Easy to create inconsistent versions.
  • Not good for many users at the same time.

A database is usually safer for long-term business data.

7. Databases help consistency

Databases support:

  • Constraints (NOT NULL, UNIQUE).
  • Relationships between tables.
  • Transactions – all or nothing updates.

This keeps business data in a consistent state.

8. SQL + Python for dashboards

You can connect Python to a database, run SQL queries and build dashboards.

import pandas as pd import sqlalchemy as sa engine = sa.create_engine("mysql+pymysql://user:pwd@host/db") df = pd.read_sql("SELECT * FROM sales", engine) # use df in a dashboard

Level 10 – Recap

  • Databases store structured business data.
  • MySQL is a relational database.
  • SQL is the main language for relational databases.
  • Use SELECT * FROM customers; to get all customers.
  • customer_id is a good primary key.
  • Databases are better than one big Excel file for consistency.
  • Constraints and transactions keep data correct.
  • SQL + Python can power business dashboards.
