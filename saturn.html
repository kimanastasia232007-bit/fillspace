Saturn · Level 6 – Debugging & Logic

Bugs, debugging with print(), syntax vs runtime errors, logical operators.

bug debugger and/or/not errors
Back to Galaxy

1. What is a bug?

A bug is an error or defect in the code that causes incorrect behavior.

  • Program crashes.
  • Wrong results.
  • Unexpected behavior.

2. Debugging with print()

One simple debugging technique is to use print() to show values at different points in the code.

def divide(a, b): print("a =", a, "b =", b) # debug print return a / b

3. Boolean expressions with and/or

Logical operators combine conditions:

  • and – both conditions must be True.
  • or – at least one condition must be True.
  • not – inverts True/False.
print(2 == 2 and 3 > 5) # False print(2 == 2 or 3 > 5) # True print(not True) # False

4. Syntax errors

A syntax error means the code is written incorrectly and Python cannot parse it.

# Missing colon - syntax error: # if x > 3 # print("Hi") # Correct: if x > 3: print("Hi")

5. Debugger tools

A debugger lets you run code step by step and inspect variables.

  • Set breakpoints.
  • Step into / over functions.
  • Watch variable values change.

6. Logical operators usage

Use logical operators to build complex conditions in if-statements and loops.

age = 20 has_id = True if age >= 18 and has_id: print("Access granted")

7. Runtime errors

A runtime error happens while the program is running, for example dividing by zero.

x = 10 y = 0 # x / y # ZeroDivisionError at runtime

8. Bad debugging practice

Randomly changing code and hoping it works is not a good debugging strategy. Instead:

  • Reproduce the bug.
  • Find where it happens.
  • Inspect variables.
  • Fix the root cause.

Level 6 – Recap

  • Bug = error in code.
  • print() is a simple debugging tool.
  • Use and, or, not for logic.
  • Syntax errors: code cannot be parsed.
  • Runtime errors: happen during execution.
  • Debuggers help inspect variables step by step.
  • Good debugging = systematic, not random changes.
