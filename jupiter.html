Jupiter · Level 5 – Files & Errors

Opening files, read modes, exceptions, try/except/finally and ValueError.

files "r" mode try/except ValueError
Back to Galaxy

1. Opening files – modes

open() takes a filename and a mode. For reading, use "r".

f = open("data.txt", "r") # read mode text = f.read() f.close()
  • "r" – read (file must exist).
  • "w" – write (creates/overwrites).
  • "a" – append.
  • "x" – create, fail if exists.

2. Closing a file

Use f.close() to close a file object.

f = open("data.txt", "r") # ... work with f ... f.close()

Or use a with-statement to close automatically:

with open("data.txt", "r") as f: text = f.read() # file is closed here

3. try / except

Use try to start a block of code that may raise an error.

try: x = int("abc") # will raise ValueError except ValueError: print("Conversion failed")

4. Handling errors with except

except handles the error if it happens inside try.

try: f = open("missing.txt", "r") except FileNotFoundError: print("File not found")

5. int() and ValueError

int() converts strings to integers but can raise ValueError.

value = "123" n = int(value) # 123 bad = "abc" # int(bad) # ValueError

6. finally block

finally runs always, whether an exception occurred or not.

f = None try: f = open("data.txt", "r") # read... except FileNotFoundError: print("File missing") finally: if f is not None: f.close()

7. Opening non-existent file in "r" mode

If you open a non-existent file in "r" mode, Python raises FileNotFoundError.

8. Why use try/except?

try/except prevents your program from crashing and lets you handle errors gracefully.

try: price = float(input("Enter price: ")) except ValueError: print("Please enter a number, not text.")

Level 5 – Recap

  • "r" mode opens a file for reading.
  • Close files with f.close() or with-statement.
  • try starts a block that might raise errors.
  • except handles the error.
  • int() can raise ValueError for invalid text.
  • finally runs whether an exception occurs or not.
  • Opening a missing file in "r" raises FileNotFoundError.
  • try/except helps avoid crashes and show user-friendly messages.
